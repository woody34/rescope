title: Configuration

Rescope is configured entirely via environment variables. There are no config files — just set variables in your shell, `.env` file, or CI environment.

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `RESCOPE_PORT` | `4600` | Port the emulator HTTP server listens on |
| `RESCOPE_PROJECT_ID` | `Ptest1234567890123456789` | Project ID returned in tokens and API responses |
| `RESCOPE_MANAGEMENT_KEY` | _(auto-generated)_ | Bearer token required for management API calls |
| `RESCOPE_SEED_FILE` | _(none)_ | Path to a JSON seed file to pre-populate users and tenants on startup |

## Example: `.env` file

```bash
RESCOPE_PORT=4600
RESCOPE_PROJECT_ID=Ptest_myapp_local
RESCOPE_MANAGEMENT_KEY=K_myapp_local_key
RESCOPE_SEED_FILE=./seed.json
```

## Seed Files

A seed file pre-populates the emulator with users and tenants on startup. The file is re-applied on `/emulator/reset`.

```json
{
  "users": [
    {
      "loginIds": ["alice@example.com"],
      "email": "alice@example.com",
      "name": "Alice",
      "status": "enabled",
      "roleNames": ["admin"]
    }
  ],
  "tenants": [
    {
      "id": "acme",
      "name": "Acme Corp",
      "domains": ["acme.com"]
    }
  ]
}
```

## Request Logging

Rescope logs every HTTP request on completion — category tag, method, path, status, latency, and (for API traffic) a truncated request body:

```
INFO rescope::http: [API] POST /v1/auth/otp/verify/email 401 2ms body={"loginId":"x@y.com","code":"000000"} resp={"errorCode":"E011003","errorDescription":"Invalid token"}
```

Lines are tagged by traffic type:

| Tag | Paths | Default visibility |
|---|---|---|
| `[API]` | `/v1/*`, `/v2/keys/*`, `/.well-known/*`, `/oauth/*` | Shown (INFO) |
| `[EMU]` | `/emulator/*`, `/health` | Shown (INFO); `/health` hidden (DEBUG) |
| `[DOCS]` | `/docs`, `/openapi.json` | Hidden (DEBUG) |
| `[UI]` | Everything else (admin UI assets) | Hidden (DEBUG) |

Set `RUST_LOG=rescope=debug` to reveal the hidden categories.

Bodies are captured only for text-like content types (JSON, form-urlencoded, `text/*`), collapsed to a single line, and truncated. Response bodies are logged only for error statuses (≥ 400). Bodies are shown unmasked — including passwords and OTP codes — because seeing exactly what your SDK sent is the point of an emulator. `Authorization` headers are never logged.

| Variable | Default | Description |
|---|---|---|
| `RESCOPE_LOG_BODY` | `1` | Set to `0` to disable body logging entirely |
| `RESCOPE_LOG_BODY_MAX` | `256` | Body truncation cap in bytes |

## Startup Banner

On startup, Rescope prints its configuration:

```
INFO rescope: ─────────────────────────────────────────────
INFO rescope:   Rescope — Descope Emulator
INFO rescope: ─────────────────────────────────────────────
INFO rescope: port=4600 project_id="Ptest1234567890123456789"
INFO rescope: management_key="K5c6e9…"
INFO rescope: ⚠️  Do not use in production
INFO rescope: ✅ Ready → http://localhost:4600
```

## Reset the emulator

You can reset all in-memory state to the seed state (or empty) at any time:

```bash
curl -X POST http://localhost:4600/emulator/reset
```

This is useful in test suite `beforeEach` hooks.
